On the Ubiquity of Python▮▮▮▮

2023-07-22

Python is everywhere: the web, the desktop, the server, the toaster—everywhere.

And I'm not the first to say it absolutely shouldn't be. The quote, I believe, is something along the lines of "Python can do everything. Just badly." In other words, a Jack of all trades, master of none. For any given task, there's a better solution—but Python works. There's been a trend lately (for about, say, 30 years) of throwing shade at Python but I think we can all agree that it has its good points. Namely... it does everything.

And that's the main problem, as I see it. Python can do everything. Badly. But because it can do so much, why would anyone learn any other language? Why program in C when it's so much more difficult to learn? And Python is so easy?

So people learn Python. Then they know Python. Then they use Python. For everything. And then people start teaching Python too. And then we have a cohort of students proficient in Python. It's a vicious cycle truly.

And most people don't consider this a problem, whether it be because they don't care, don't see it as important, or actually like Python.

I could rant forever on how bad Python is, but I wouldn't have much original to say that hasn't already been written far more eloquently than I ever could have by some strangers on the ol' interwebs.

Surely if everyone's learning the same one language for everything it may as well be one that's at least a little better than Python right? Take Julia for example. A simple syntax akin to that of Python, yet with such a cleaner type system, decent dependency management, miles better performance, macros—the list goes on.

So please. Move to Julia. Julia can do everything too. And better.
