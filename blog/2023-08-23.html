PiCMCA▮▮▮▮

2023-08-23

The Pi Compute Module Cardbus Adapter.

Introduction

The PiCMCA is a Cardbus (PCMCIA) module that uses the USB lanes of the Cardbus interface as a low-latency network connection between the host system and a Raspberry Pi.

This allows for the host to act as a thin client for the Raspberry Pi, thus allowing vintage computers to act as modern computing devices.

Initially I had intended to use a Compute Module 4 for this, since this would allow the board to be reused for any module using the CM4 standard, but unfortunately, at the time of writing, these are relatively difficult and expensive to obtain. Instead I am using a Raspberry Pi Zero, though the same could be achieved with, for example, a Radxa Zero instead.

Why?

I am the happy owner of a Thinkpad x61, which is now... 14 years old. It is, of course, beginning to show its age, with its fans loud as a microwave, and a battery with the lifespan of an elderly mayfly.

For all that, it is still in excellent condition, so I wanted to give it another chance at life. Thus I had the idea to put a computer inside the computer.

Plan

The Pi would use the Thinkpad SSD for storage using some such magic as NFS or SSHFS etc., since the SD card is obviously unsuitable for anything more than the OS. Ideally I'd have the entire root filesystem on the Thinkpad using PXE/netbooting, but alas my proficiency in wizardry extends only so far. We might be able to have the boot partition on the SD card then mount root as NFS?

It's quite a mess to have the files on the client but this isn't Plan9 unfortunately so we can't just set each as CPU server and File server. I was actually considering setting up Plan9 but I felt like that would have hindered usability to the point where the whole project would have been more of a meaningless waste of time and money than it already was.

The Zero would connect to the PCB by the USB-C and HDMI ports, and would also use the GPIO pins for a headphone jack.

In my research I considered a number of possible routes:

  • SSH X forwarding
  • Waypipe
  • XDMCP

Implementation

In the end I went with XDMCP for low latency, and so that I wouldn't also need to run an X server or Wayland session on the Thinkpad as well as on the Pi. The lack of encryption is a non-issue since only the Pi and the Thinkpad are going to be using the usb network connection. For storage NFS presented itself as a decent solution. I'm partial to anything Sun so it was a fairly easy decision (I'd have loved to have done this on a Tadpole Sparcbook. Alas I do not have one and they don't have cardbus slots anyway).

To test that the theoretical USB interface would work, I first connected the Pi over regular USB then set up XDMCP. I then designed a PCB to mount the Pi on, and designed and 3D printed a frame using Blender and my trusty Ender 3 to protect the components and provide better robustness. I used PETG filament for its high heat resistance and durability, but a more environmentally friendly solution could be to use PLA instead.

Evaluation

The project definitely works, but the Pi Zero is perhaps not as powerful as I had thought. In hindsight the CM4 would have been a much better choice, but alas they remain unobtainable at reasonable prices. For an even older laptop like the Thinkpad 235 (with its 3 cardbus slots and standard replaceable batteries) the Zero would be a significant upgrade however, being able to run modern applications such as web browsers, at not insufferable performance, therefore I shall consider this project a huge success!
